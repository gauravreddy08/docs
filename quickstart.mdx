## Setup your development environment

<Steps>
  <Step title="Start the API server">
    Use Docker Compose to quickly start the Vector Database API:
    
    ```bash
    docker compose up -d
    ```
    
    <Check>
    The API will be available at `http://localhost:8000` with interactive documentation at `/docs`.
    </Check>
  </Step>

  <Step title="Verify the server is running">
    Check the health endpoint to ensure everything is working:
    
    ```bash
    curl http://localhost:8000/health
    ```
    
    You should see a response like:
    ```json
    {
      "status": "healthy",
      "message": "Running", 
      "version": "1.0"
    }
    ```
  </Step>
</Steps>

## Create your first library

A library is the top-level container for your documents and chunks. It also defines which indexing algorithm to use.

<CodeGroup>

```bash cURL
curl -X POST 'http://localhost:8000/v1/libraries/' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "my-first-library",
    "index_type": "linear",
    "metadata": {
      "description": "My first vector database library"
    }
  }'
```

```python Python
import requests

response = requests.post('http://localhost:8000/v1/libraries/', json={
    "name": "my-first-library", 
    "index_type": "linear",
    "metadata": {
        "description": "My first vector database library"
    }
})

library = response.json()
library_id = library['id']
```

</CodeGroup>

<ResponseExample>

```json Response
{
  "id": "123e4567-e89b-12d3-a456-426614174000",
  "name": "my-first-library",
  "created_at": "2024-01-15T10:30:00Z",
  "metadata": {
    "description": "My first vector database library"
  },
  "index_type": "linear",
  "index_params": null,
  "documents": []
}
```

</ResponseExample>

<Tip>
Save the `id` from the response - you'll need it for the next steps!
</Tip>

## Add your first chunk

Chunks are pieces of text that get converted to vector embeddings for similarity search. When you create a chunk without specifying a `document_id`, a new document is automatically created.

<CodeGroup>

```bash cURL
curl -X POST 'http://localhost:8000/v1/libraries/{library_id}/chunks/' \
  -H 'Content-Type: application/json' \
  -d '{
    "text": "The quick brown fox jumps over the lazy dog",
    "metadata": {
      "category": "example",
      "language": "en"
    }
  }'
```

```python Python
chunk_response = requests.post(f'http://localhost:8000/v1/libraries/{library_id}/chunks/', json={
    "text": "The quick brown fox jumps over the lazy dog",
    "metadata": {
        "category": "example", 
        "language": "en"
    }
})

chunk = chunk_response.json()
print(f"Created chunk: {chunk['id']}")
```

</CodeGroup>

Let's add a few more chunks to make search more interesting:

<CodeGroup>

```bash cURL
# Add more sample chunks
curl -X POST 'http://localhost:8000/v1/libraries/{library_id}/chunks/' \
  -H 'Content-Type: application/json' \
  -d '{"text": "Machine learning algorithms process data efficiently", "metadata": {"category": "tech"}}'

curl -X POST 'http://localhost:8000/v1/libraries/{library_id}/chunks/' \
  -H 'Content-Type: application/json' \
  -d '{"text": "Natural language processing enables text understanding", "metadata": {"category": "tech"}}'
```

```python Python
# Add more sample chunks
sample_chunks = [
    {

    },

        "text": "Natural language processing enables text understanding",

    }

for chunk_data in sample_chunks:
    requests.post(f'http://localhost:8000/v1/libraries/{library_id}/chunks/', json=chunk_data)
```

</CodeGroup>

## Build the index

For some index types like IVF, you need to build the index before searching. Linear indexes don't require this step, but it's a good practice to call it anyway:

<CodeGroup>

```bash cURL
curl -X POST 'http://localhost:8000/v1/libraries/{library_id}/index'
```

```python Python
index_response = requests.post(f'http://localhost:8000/v1/libraries/{library_id}/index')
print("Index built successfully!")
```

</CodeGroup>

## Search for similar content

Now you can search for chunks similar to your query text:

<CodeGroup>

```bash cURL
curl -X POST 'http://localhost:8000/v1/libraries/{library_id}/search' \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "fast algorithms and data processing", 
    "k": 2
  }'
```

```python Python
search_response = requests.post(f'http://localhost:8000/v1/libraries/{library_id}/search', json={
    "query": "fast algorithms and data processing",
    "k": 2
})

results = search_response.json()
for result in results['results']:
    print(f"Score: {result['score']:.3f} - Text: {result['chunk']['text']}")
```

</CodeGroup>

## Search with filters

You can also search with metadata filters to narrow down results:

<CodeGroup>

```bash cURL
curl -X POST 'http://localhost:8000/v1/libraries/{library_id}/search' \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "algorithms",
    "k": 5,
    "filters": {
      "category": "tech"
    }
  }'
```

```python Python
filtered_search = requests.post(f'http://localhost:8000/v1/libraries/{library_id}/search', json={
    "query": "algorithms",
    "k": 5, 
    "filters": {
        "category": "tech"
    }
})

results = filtered_search.json()
print(f"Found {len(results['results'])} results with category='tech'")
```

</CodeGroup>
